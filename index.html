§ 15 min

Hermes 是什么

不是 LangChain、不是 AutoGen、不是又一个聊天 bot。Hermes 是个持续学习的个人 agent 框架。

它解决了什么

绝大多数 agent 框架的问题是用完即忘。你今天问它怎么写 Python 函数，它把代码给你；明天你再问，它又从零开始解释。

Hermes 的差异点是这 3 个：

  • 长期记忆：跨会话记住你是谁、你在做什么、你的项目状态。用 SQLite FTS5 做全文检索。
  • 技能沉淀：复杂任务完成后自动写一个 SKILL.md 存到 ~/.hermes/skills/。下次类似问题直接调技能，不重新发明。
  • 行为改进：技能被反复调用时会自我改进。错过的任务会变成新的技能。

和其他框架的差异（简单对比）

能力HermesLangChainAutoGen
跨会话记忆✓ 内置基础无
技能自动创建✓ 内置无无

执行环境（SSH/Docker/Modal）6 个后端自己接有限

一次完整调用会发生什么

你在飞书发一条语音："让机械臂挥挥手"。Hermes 内部经历这些步骤：

飞书 webhook → ASR 转写 → 记忆检索 → prompt 组装 → LLM（DeepSeek） → 工具调用 → TTS 回复 → 飞书回复

伪代码（这是它最核心的样子）：

# 简化版，真实代码在 run_agent.py（5,461 行）
def handle_message(text, user):
    # 1. 拉记忆
    context = memory.recall(user, text)            # SQLite FTS5

    # 2. 装工具表（默认 + 激活的 skill）
    tools = registry.active_tools(user.profile)

    # 3. 调 LLM
    response = llm.chat(context + text, tools=tools)

    # 4. 如果 LLM 要调工具，循环跑
    while response.tool_calls:
        for call in response.tool_calls:
            result = tools[call.name].run(call.args)   # ★ 这里能调 ros2
        response = llm.chat(... + results, tools=tools)

    # 5. 回写记忆
    memory.sync_turn(user, text, response.text)

    return response.text

你注意第 4 步——这里就是机器人的接入点。LLM 说"我要调一个叫 ros2_topic_publish 的工具"，框架就真的去调 ROS2。这一行就是关键。

一句话总结：Hermes = 大模型 + 长期记忆 + 工具调度 + 技能沉淀 + 20 个消息平台。我们要用它，就是因为"长期记忆"和"技能沉淀"这两个东西能让 agent 越用越聪明——这是其他框架没有的。
§ 210 min

怎么把机器人接进来

4 种接法，复杂度从低到高。我们大概率选 1-2 种就够了。

方式 A · 让 Hermes 跑 ros2 命令（最快，5 分钟）

Hermes 自带 terminal 工具。LLM 决定要执行什么 shell 命令，Hermes 就跑：

$ ros2 topic pub /sgr532/joint1_position_controller/command \
        std_msgs/msg/Float64 "data: 1.57" --once

# Hermes 看到这条命令的 stdout 退出码 0 = 成功
# 它会把"机械臂挥手了"写进回复发给飞书

优点：零代码，5 分钟接上。
缺点：没有结构化校验，LLM 拼错命令就崩；不适合"高频"调用（每次 LLM 都要决定拼命令）。

方式 B · 写一个 SKILL.md 告诉 agent 怎么用（推荐，30 分钟）

Hermes 加载 optional-skills/holo-robotics-ros2-bridge/SKILL.md，里面写：

---
name: holo-robotics-ros2-bridge
description: 用 ros2 CLI 控制 sagittarius 机械臂。
  适用于单次发布 joint 位置、查询 topic、调用 service。
  高频任务请用 MCP server。
prerequisites:
  commands: [ros2]
metadata:
  hermes:
    tags: [robotics, ros2, sagittarius]
---

# sagittarius 机械臂 · ROS2 控制

## 关节名称
joint1, joint2, joint3, joint4, joint5, joint6
对应控制器: /sgr532/joint{N}_position_controller/command

## 常见动作

### 让机械臂挥手（关节 1 摆动）
$ ros2 topic pub /sgr532/joint1_position_controller/command \
    std_msgs/msg/Float64 "data: 1.57" --once
$ sleep 1
$ ros2 topic pub /sgr532/joint1_position_controller/command \
    std_msgs/msg/Float64 "data: -1.57" --once

LLM 读到这个 SKILL.md，就知道"挥手"该执行什么。等于给 agent 写了一份操作手册。

方式 C · 起一个 MCP server（最干净，半天到一天）

MCP（Model Context Protocol）是给 LLM 用结构化 schema 调外部服务的标准。Hermes 是 MCP client，启动后自动发现 server 注册的工具。

骨架代码：

# ros2_bridge_server.py
from mcp.server import Server
import rclpy
from sensor_msgs.msg import JointState

app = Server("ros2-bridge")

@app.tool()
def move_joint(joint_name: str, position: float) -> str:
    """把指定关节移动到目标位置（弧度）"""
    rclpy.init()
    node = rclpy.node.Node("holo_bridge")
    pub = node.create_publisher(Float64,
        f"/sgr532/{joint_name}_position_controller/command", 10)
    pub.publish(Float64(data=position))
    return f"已发布 {joint_name} → {position} rad"

@app.tool()
def get_joint_states() -> dict:
    """读当前所有关节位置"""
    # ... 订阅 JointState 一次性读
    return {"joint1": 0.0, ...}

if __name__ == "__main__":
    app.run()

在 ~/.hermes/config.yaml 加：

mcp_servers:
  ros2_bridge:
    command: python3 /home/leo/aquarius-psyche/holo/mcp/ros2_bridge_server.py
    description: ROS2 桥 · 控制 sagittarius 机械臂

Hermes 启动时自动发现 move_joint / get_joint_states，LLM 就有结构化工具调。优点：schema 校验、类型安全、可以多机部署。缺点：要起一个进程。

方式 D · 改 core（不推荐）

Hermes 的 run_agent.py / cli.py 是核心循环。AGENTS.md 明说"核心是窄腰，所有新能力放边缘"。改 core 会污染 prompt cache、拖累所有用户。**只有真正通用的改动才改 core。**

怎么选？看这个表

A · 直接跑命令B · SKILL.mdC · MCP serverD · 改 core
接入时间5 分钟30 分钟半天-1 天数天 + 风险
结构化校验无无有（schema）有
高频调用友好差中好好
适合验证 / Demo教学场景 / 偶尔操作实时控制 / 生产通用改进
可逆性不写代码删文件关进程回滚 PR
我的建议：第一周先用 B（SKILL.md）让你能跟它"对话控制机械臂"；第二周用 C（MCP）做实时闭环控制。A 用作 fallback，D 不碰。

一个具体的场景走一遍

学生问："让机械臂从 home 位挥手"

  1. Hermes 飞书收到消息 → ASR → "让机械臂从 home 位挥手"
  2. 记忆检索：发现学生上节课学了 joint state
  3. prompt 组装：注入系统提示 + 学生记忆 + 课程库检索 + holo-robotics-ros2-bridge 技能
  4. LLM 决定：调 MCP 工具 move_joint("joint1", 1.57) → 调 move_joint("joint1", -1.57) → 调 move_joint("joint1", 0.0)
  5. MCP server 通过 ros2 发布 Float64 到 joint controller
  6. Gazebo 里机械臂真的挥了一下
  7. LLM 生成文本回复："挥完了 ✅"
  8. 记忆写入：学生今天学会了"挥手 = 3 个 move_joint 串联"
  9. TTS → 飞书回复

整个链路里，LLM 是大脑，MCP server 是脊髓，ROS2 是神经，Gazebo 是肌肉。Hermes 把大脑到脊髓的部分都打通了——我们要做的就是补脊髓和神经。

每门课怎么落进 agent

每门课变成 两件东西：

  1. 一个 RAG 索引：把课程文档（PDF / Markdown / 笔记）拆成 chunk，灌进 ChromaDB。Hermes 调 memory 工具时自动检索。
  2. 一组机器人动作：每节课里有"会动的概念"，写进 SKILL.md 让 agent 触发。MCP server 真去调机器人。
